# Task Management API

FastAPI • PostgreSQL • Docker • JWT Authentication • Pytest • 90%+ Test Coverage

Production-style backend API for managing user tasks with authentication, database persistence, and automated testing.

This project demonstrates a typical backend architecture used in modern backend systems, including layered architecture (routers, services, models, schemas), JWT authentication, database migrations, and containerized development environments.

---

## Features

- User registration and login using JWT authentication
- Task CRUD operations (create, read, update, delete)
- User-level data isolation (users can only access their own tasks)
- Pagination support
- PostgreSQL database with SQLAlchemy ORM
- Alembic database migrations
- Dockerized development environment
- Automated API tests with Pytest
- Test coverage reporting (>90%)

  models/         database models

tests/            automated API tests

Dockerfile  
docker-compose.yml  

The project follows a layered architecture where routers remain thin and business logic is handled in service modules.

## Running the project

Start the API and database with Docker:

docker compose up --build

API documentation will be available at:

http://localhost:8000/docs

Swagger UI is automatically generated by FastAPI.

---

## Running Tests

Execute the automated test suite:

docker compose run tests

---

## Test Coverage

Generate coverage report:

docker compose run tests pytest --cov=app

Coverage currently exceeds 90%, ensuring the core API functionality is well tested.

## Example API Workflow

Register user

POST /auth/register

Example body:

{
  "email": "user@example.com",
  "password": "test123"
}

Login

POST /auth/login

Returns a JWT token used for authenticated requests.

Create task

POST /tasks

Example body:

{
  "title": "Example Task",
  "description": "Task created via API",
  "completed": false
}

Retrieve tasks

GET /tasks

Update task

PUT /tasks/{task_id}

Delete task

DELETE /tasks/{task_id}
